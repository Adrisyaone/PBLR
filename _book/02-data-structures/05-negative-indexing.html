25  Negative Indexing

The Problem

Your supervisor tells you: “Actually, participant #1 was a pilot test entry from before the study officially started — exclude that one row from all your calculations, but keep the other 99.” You know how to pull specific elements out of a vector using positive indexing, but now you need the opposite: everything except one particular position.

Your Mission

Exclude one or more elements from a vector by position, rather than selecting elements to include.

Think Before Coding

  • If positive numbers inside [ ] mean “give me these positions,” what might the opposite sign mean?
  • If you needed to exclude both the 1st and 2nd elements, how might you extend a single negative index to several?

25.1 The R Concept

Putting a minus sign in front of a position inside [ ] tells R to exclude that position, returning everything else:

x[-position]
ages <- c(46, 53, 59, 49, 61)
ages[-1]        # everything except the 1st element
ages[-c(1, 2)]  # everything except the 1st and 2nd elements

This mirrors positive indexing exactly, just inverted — and it composes with everything you learned on the last page, including ranges: ages[-(1:5)] excludes the first five elements.

25.2 Demonstration

ages <- c(46, 53, 59, 49, 61, 18, 62, 68, 33, 36)

ages[-1]
[1] 53 59 49 61 18 62 68 33 36
ages[-c(1, 2)]
[1] 59 49 61 18 62 68 33 36
ages[-(1:3)]
[1] 49 61 18 62 68 33 36

25.3 Explore

Try mixing signs: ages[c(-1, 2)]. What happens? R will refuse and produce an error — predict what that error might say before you run it, based on the idea that “include position 2” and “exclude position 1” are contradictory instructions to combine in one call.

25.4 Solve the Problem

participants <- read.csv("data/participants.csv")

# Exclude participant 1 (the pilot test entry) from the age calculation
valid_ages <- participants$age[-1]

length(valid_ages)
[1] 99
mean(valid_ages, na.rm = TRUE)
[1] 47.36735

25.5 Interpret the Result

valid_ages now holds 99 values instead of 100, correctly excluding the pilot test entry, and the mean age recalculated on this corrected set is 47.4 years (rounded) — very close to, but not identical to, the full-sample mean from Level 1, since removing one participant necessarily shifts the average slightly.

Common Mistakes

  • Mixing positive and negative indices in one call. x[c(-1, 2)] produces an error (can't mix positive and negative subscripts) — you must choose one approach or the other for a single indexing operation.
  • Forgetting parentheses around a range you want to negate. -1:3 is interpreted as “the sequence from −1 to 3,” not “exclude positions 1 through 3” — you need -(1:3) with explicit parentheses to negate the whole range.
  • Assuming negative indexing modifies the original vector. ages[-1] returns a new vector with one element excluded; it does not delete anything from ages itself unless you explicitly reassign the result back: ages <- ages[-1].

25.6 Practice Problem

Using participants$bmi, create a vector bmi_excluding_first_three that excludes the first three participants’ BMI values, using negative indexing with a range.

25.7 Challenge Problem

Your supervisor now says participants #1 and #100 were both pilot test entries — exclude both from participants$age in a single line of negative indexing, and compute the mean of the remaining 98 ages.

25.8 Key Takeaways

  • A negative sign inside [ ] excludes that position (or positions) rather than selecting it.
  • Positive and negative indices cannot be mixed in a single indexing call.
  • Indexing (positive or negative) returns a new vector — it doesn’t modify the original unless you explicitly reassign the result.

Knowledge Check

  1. Multiple choice. What does x[-2] return for x <- c(10, 20, 30)?

    1. 20
    2. c(10, 30)
    3. c(20, 30)
    4. An error

  2. True or false. x[c(-1, 2)] is valid R code that excludes position 1 and includes position 2.

  3. Predict the output.

    x <- c(5, 10, 15, 20, 25)
x[-(1:2)]

  4. Find the error.

    x[-1:3]
Show answers
  1. b) c(10, 30) — position 2 (20) is excluded.
  2. False. Mixing positive and negative indices in the same call raises an error.
  3. [1] 15 20 25
  4. Without parentheses, -1:3 is read as the sequence from −1 to 3, not “exclude 1 through 3.” It should be x[-(1:3)].
