129  Level 10 Capstone: Automating Five Clinic Files

The Problem

This level’s opening mission, finally in front of you: data/clinics/ contains five separate files — one per county’s clinic — each with the exact same column structure as participants.csv. Reading, checking, and combining them by hand would mean writing (and re-running) the same three lines five separate times. Your professor wants one script that handles all five files in a single run, however many files there happen to be.

Your Mission

Use list.files() to discover every clinic file automatically, then loop through them, reading and combining all five into one dataset with a single script.

Think Before Coding

  • From Level 3, what did read.csv() need as its argument — a fixed, hardcoded filename, or could that filename come from a variable instead?
  • If you had a vector containing all five file paths, would a for loop (Page 1) be a natural way to read each one in turn?

129.1 The R Concept

list.files() returns a character vector of filenames found in a directory, optionally filtered by pattern — exactly what’s needed to discover “however many files happen to exist” without hardcoding a count:

file_list <- list.files("data/clinics", pattern = "\\.csv$", full.names = TRUE)

Combined with a for loop and result accumulation (Page 2), this reads and combines every discovered file automatically:

all_data <- list()   # a list to accumulate each file's data frame

for (i in seq_along(file_list)) {
  all_data[[i]] <- read.csv(file_list[i])
}

combined <- do.call(rbind, all_data)   # stack all data frames into one

129.2 Demonstration

clinic_files <- list.files("data/clinics", pattern = "\\.csv$", full.names = TRUE)

clinic_files
[1] "data/clinics/clinic_cedarfalls.csv"  "data/clinics/clinic_millbrook.csv"
[3] "data/clinics/clinic_oakford.csv"     "data/clinics/clinic_riverdale.csv"
[5] "data/clinics/clinic_springvale.csv"

list.files() found all five clinic files automatically — if a sixth clinic file were added tomorrow, this same line would discover it too, with no code changes needed.

129.3 Explore

Try length(clinic_files). Confirm it returns 5, matching the number of counties in the original study — and consider what would happen to this same code if a clinic closed (4 files) or a new one opened (6 files): nothing in the loop itself would need to change.

129.4 Solve the Problem

clinic_files <- list.files("data/clinics", pattern = "\\.csv$", full.names = TRUE)

all_clinics <- list()

for (i in seq_along(clinic_files)) {
  all_clinics[[i]] <- read.csv(clinic_files[i])
}

combined_participants <- do.call(rbind, all_clinics)

nrow(combined_participants)
[1] 100
table(combined_participants$county)
Cedar Falls   Millbrook     Oakford   Riverdale  Springvale
         20          21          17          16          26
round(mean(combined_participants$age, na.rm = TRUE), 2)
[1] 47.43

100 total rows, county counts of 20/21/17/16/26, and a mean age of 47.43 — every one of these exactly matches the original participants.csv used since Level 1, confirming the five separate clinic files, read and combined automatically, reconstruct the identical dataset.

129.5 Interpret the Result

The automation succeeded completely: a single script, running once, correctly discovered, read, and combined all five clinic files into a dataset statistically identical to the original — same row count (100), same county breakdown, same mean age (47.43). If your team’s five clinics send updated files next month, this exact same code would run again with zero changes, reading whatever files exist in data/clinics/ at that time.

Common Mistakes

  • Hardcoding file names instead of using list.files(). Writing five separate read.csv("data/clinics/clinic_cedarfalls.csv")-style lines defeats the purpose of automation — if a clinic’s filename or count changes, hardcoded code needs manual updates every time; list.files() adapts automatically.
  • Forgetting full.names = TRUE. Without it, list.files() returns just the filenames (like "clinic_cedarfalls.csv") without the folder path — read.csv() would then fail to find the file unless your working directory (Level 1) happened to already be data/clinics/.
  • Using rbind() directly on a list instead of do.call(rbind, list). rbind(all_clinics) (without do.call()) doesn’t correctly stack a list of data frames — do.call(rbind, all_clinics) is what correctly applies rbind() across every element of the list at once.

129.6 Practice Problem

Modify the loop to also print each file’s row count as it’s read (e.g., "clinic_oakford.csv: 17 rows"), confirming each file’s contribution before they’re all combined.

129.7 Challenge Problem

Add a validation step after combining: use sum(is.na(combined_participants$age)) to confirm the combined dataset has exactly 6 missing ages, matching the original participants.csv exactly. Then write a general-purpose function (Level 9) called read_and_combine that takes a folder path and file pattern as arguments, returning the combined data frame — so this entire capstone becomes reusable for any future folder of identically structured files, not just data/clinics/.

129.8 Key Takeaways

  • list.files(folder, pattern = ..., full.names = TRUE) discovers files automatically, adapting to however many exist without hardcoding names or counts.
  • Combining a loop (Page 1), result accumulation into a list (Page 2), and do.call(rbind, list) automates reading and stacking any number of identically structured files into one dataset.
  • True automation means the same code keeps working correctly even as the underlying number or names of files change — exactly the property that made this capstone’s approach worth building, instead of hardcoding five separate read.csv() calls.
