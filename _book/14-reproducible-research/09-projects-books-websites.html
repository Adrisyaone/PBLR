170  Quarto Projects: Books and Websites

The Problem

You’ve now written parameterized single reports (Page 7), but this entire course you’ve been working through — over 100 pages, organized into numbered levels with a shared sidebar and search — is clearly something bigger than one .qmd file. Your professor asks: what’s actually different about a Quarto project, and how does it turn a folder of separate .qmd files into one unified book like this one?

Your Mission

Understand the difference between a single Quarto document, a Quarto website, and a Quarto book, and recognize which type this course itself is.

Think Before Coding

  • A single .qmd file with its own YAML front matter (Pages 1-8) versus a _quarto.yml at a project’s root (Page 2) — which one applies to just one page, and which applies to every page in a folder?
  • This course has 15 numbered levels with a shared sidebar, search bar, and consistent theme across every single page. Would a collection of unrelated, separately-rendered .qmd files naturally have that in common?

170.1 The R Concept

A Quarto project is a folder containing a _quarto.yml file that ties multiple .qmd files together into one coherent output, rather than each rendering independently. The project: type: field determines what kind of unified output results:

# A Quarto website project
project:
  type: website
website:
  title: "My Site"
  navbar:
    left:
      - href: index.qmd
      - href: about.qmd
# A Quarto book project - exactly what this course is
project:
  type: book
book:
  title: "R for Public Health"
  chapters:
    - index.qmd
    - 01-getting-started/00-overview.qmd
    - 01-getting-started/01-what-is-r.qmd

A website organizes independent pages with shared navigation; a book organizes sequential chapters (with numbering, a defined reading order, and often PDF/EPUB output) — this course’s own _quarto.yml (Page 2) uses type: book specifically because its 15 levels are meant to be read in order, not browsed as unrelated pages.

170.4 Solve the Problem

# A minimal Quarto BOOK project structure, from scratch:

  type: book

  title: "My Public Health Report Series"
  chapters:
    - index.qmd
    - chapter1-methods.qmd
    - chapter2-results.qmd
    - chapter3-discussion.qmd

format:
  html:
    theme: flatly
  pdf:
    documentclass: report
Running `quarto render` on this project produces a single, unified
multi-chapter output: a website with consistent navigation between
chapters (in HTML), AND a single combined PDF book with chapters in
the specified order - both generated from the same 4 separate .qmd
files, exactly the same mechanism producing this entire course you've
been reading, just at a much smaller scale (4 chapters instead of 15
levels' worth of lessons).

170.5 Interpret the Result

A Quarto project’s _quarto.yml is what elevates a folder of individually-useful .qmd files into one coherent, navigable whole — shared theme, shared table of contents structure, and (for a book) a defined chapter order and combined PDF/EPUB output. This course itself is the single largest example of this idea you’ve encountered: every one of the 100+ separate .qmd files you’ve read was rendered as part of one unified book project, not as 100+ disconnected, individually-rendered pages.

Common Mistakes

  • Confusing a website project (independent, browsable pages) with a book project (sequential, ordered chapters). A website’s navbar: structure suits unrelated or loosely-related pages; a book’s chapters: list (with part: groupings, exactly like this course’s 15 levels) suits genuinely sequential content meant to be read in order.
  • Forgetting new chapter files must be added to _quarto.yml’s chapters: list. Simply creating a new .qmd file inside a project folder does not automatically include it in the book or website — every chapter must be explicitly listed, exactly as this course has done throughout Levels 1-14’s own construction.
  • Rendering individual .qmd files directly instead of rendering the whole project. Inside a book or website project, rendering a single chapter file in isolation may lose the shared navigation, numbering, and cross-references (Page 5) that only appear correctly when the entire project is rendered together.

170.8 Key Takeaways

  • A Quarto project (marked by a _quarto.yml file) ties multiple .qmd files together into one coherent, unified output, rather than rendering each file independently.
  • project: type: website suits independent, navigable pages; project: type: book (used by this entire course) suits sequential, ordered chapters, often with part: groupings and combined PDF/EPUB output.
  • Every chapter file must be explicitly listed in _quarto.yml’s chapters: list to be included in the rendered project — creating a .qmd file alone isn’t enough.

## The R Concept

```yaml
# A Quarto website project
project:
  type: website
website:
  title: "My Site"
  navbar:
    left:
      - href: index.qmd
      - href: about.qmd
```

```yaml
# A Quarto book project - exactly what this course is
project:
  type: book
book:
  title: "R for Public Health"
  chapters:
    - index.qmd
    - 01-getting-started/00-overview.qmd
    - 01-getting-started/01-what-is-r.qmd
```

A **website** organizes independent pages with shared navigation; a **book** organizes sequential chapters (with numbering, a defined reading order, and often PDF/EPUB output) — this course's own `_quarto.yml` (Page 2) uses `type: book` specifically because its 15 levels are meant to be read in order, not browsed as unrelated pages.
